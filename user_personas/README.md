# User Personas  

## Purpose
The purpose of this directory is to contain the key stakeholders involved in this project and their contributions. These personas give information about the people interacting with the software, their role, goals, and pain points.  Understanding the personas will help to enhance the software, interaction between personas, and make sure that expectations are delivered. This information helps:  

- **Developers** – Understand the expectations, workflows, and communication needs.  
- **Owner** – Ensure that the software is what was needed.  
- **End Users** – Identify user needs to improve on.  

## Contents  
Each user persona includes:  
- **Name**
- **Role** 
- **Demographic**
- **Goals** - their needs. 
- **Pain Points** – challenges  
- **Scenario** – how they interact with the software.  

## How to Use This Information  
- **Development:** Enhance the website to keep up with user needs  
- **Interaction between teams:** communication to ensure good workflow and to meet owner's needs
- **Testing:** Ensure workflows and functionalities work for realistic test cases.  
- **Documentation:** consistent and readable documentation to keep developers informed about code segments

By defining these personas, the project remains user-focused, developer-friendly, and aligned with stakeholder goals.
